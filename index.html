Correlations between Stock Performance and Google Search Trends
¶

James Riley Wilburn
¶

The S&P 500 is a collection of 500 companies with the highest market capitalization. Market capitalization (market cap) is the price of a given company's stock multiplied by the total number of shares of that stock that are publicly traded. Market cap is used as a measure for the general value of an entire company. In general, the S&P 500 is a collection of the 500 most currently valuable companies. More details on the S&P 500 can be found here. The companies in the S&P 500 were chosen for this tutorial because there is generally a large amount of historical data available for companies of this size. It is also useful to know how to analyze financial in general because a significant portion of stock trading is now done algorithmically. Knowing how to interact with market data is a necessary skill to creating predictive market models.

The primary goal of this tutorial is identify potentially interesting correlations between financial data and various terms associated with each company. Search trend data is gathered from Google, while the financial data is sourced from Kaggle.

Gathering the Data
¶

Five years of S&P 500 daily data, including market open price, market close price, daily high, daily low, and daily volume (# of shares) traded was obtained from this source on Kaggle. This data already in a nice comma separated value (CSV) format. A CSV file of the member companies of the S&P 500, their stock ticker, and their general industry category from DataHub is also used to categorize the companies by industry. As the S&P 500 changes membership occasionally and the two CSV files have differing ages, the sets of companies they contain are slightly different. For the purposes of this tutorial, the intersection of these two sets will be used.

Google Search Data
¶

Unfortunately, google search trend data is not easily gatherable with python. At the time of writing, the python Google Trends API library pytrends is not functioning properly, possibly because the Google Trends API has been updated more recently than that library. Instead I had to use the google-trends-api NodeJS library. NodeJS is a platform that allows execution of JavaScript on the server side, rather than solely in a browser. The following code gathers Google Trends data from their unofficial API, so the following instructions may be incorrect in the near future.

// imports
const googleTrends = require("google-trends-api");
const parse = require("csv-parse/lib/sync");
const fs = require("fs");
const path = require("path");

// gather search data from Google Trends API
async function searchData(word) {
  let results = await googleTrends.interestOverTime({
    keyword: word,
    startTime: new Date("2013-02-08"),
    endTime: new Date("2018-02-08")
  });
  try {
    return {
      word: word,
      timeline: JSON.parse(results).default.timelineData.map(p => ({
        time: p.time,
        value: p.value[0]
      }))
    };
  } catch {
    console.error(results);
  }
}

// gather and tag Google Trend data for a list of words
async function gather(list) => {
  var results = [];
  for (i in list) {
    const word = list[i];
    await sleep(500);
    const data = await searchData(word);
    results.push(data);
    console.error(word);
  }
  return results;
};

// Sleep for ms milliseconds
function sleep(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

// Read S&P 500 members list as a string
const wordData = fs.readFileSync(
  path.join(__dirname, "constituents-financials_csv.csv"),
  "utf8"
);

// Parse S&P 500 members list into an object
const wordList = parse(wordData, {
  columns: true,
  skip_empty_lines: true
});

//Split S&P members list into its component columns
const symbols = wordList.map(x => x.Symbol);
const names = wordList.map(x => x.Name);
const sectors = [...new Set(wordList.map(x => x.Sector))];


// Gather and write Google Trends data to json files
(async () => {
  fs.writeFileSync("symbols.json", JSON.stringify(await gather(symbols)));
  fs.writeFileSync("names.json", JSON.stringify(await gather(names)));
  fs.writeFileSync("sectors.json", JSON.stringify(await gather(sectors)));
})();

The above code reads the S&P 500 companies list, then for each stock symbol, company name, and industry name in the list, it gathers Google Search trend data for the same 5 year period as the financial data. It outputs the gathered data as JSON documents. JSON is a textual dataformat which is easily human readable and more structured than CSV.

search_trends_symbol = pd.concat(
    map(
        lambda x: x[1],
        pd.read_json("symbols.json", orient="columns").apply(
            lambda row: pd.DataFrame.from_records(
                # Iterate over all of the timeline data
                map(
                    lambda t: { # t is a timeline row of the form {time, value}
                        "symbol": row['word'],
                        # Converts the unix timestamp to a real datetime
                        "timestamp":pd.to_datetime(t['time'], unit='s').floor('d'),
                        "frequency":t['value']
                    },
                    row.timeline,
                ) , 
                index=["symbol", "timestamp"]
            ) if len(row.timeline) else None, 
            axis=1,
        ).iteritems(),
    )
)

The above code processes the symbols JSON document and formats so that it can be digested into a pandas DataFrame. Essentially, for each of the stock symbols in the improper DataFrame above, it creates a DataFrame with columns timestamp, frequency, search_term, indexd on search_term and timestamp. Then it concatenates each of the new DataFrames into a complete DataFrame. The if len(row.timeline) else None ensures that the DataFrame concatenation does not break when there is no timeline data for the search term. This happens when the term is too specific for google to record its trends, which happens for some of the longer company names. The timestamps are rounded to the nearest day so that they can be compared for exact equality. This greatly simplifies processing later.

Preprocessing the Data
¶

Before the raw data is ready for analysis, it must be preprocessed. In this case, the appopriate terms and frequency data must be mapped to the financial data in order to do the correlations between them. In order to match the timestamps between the stock ticker data and the search trends data, the stock data weeks must be condensed into one data point per week per ticker from the original daily datapoints. This will also tend to reduce noise in the data and minimize weekly fluctuations.

The above code joins all of the search data across symbols, company names, and sectors using the company DataFrame which contains a mapping between all of those values. For those that know SQL, the above is equivalent to the following:

SELECT 
    symbols.symbols as symbol, 
    symbols.timestamp as timestamp, 
    companies.name as name, 
    companies.sector as sector,
    companies.market_cap as market_cap,
    symbols.frequency as symbol_frequency, 
    names.frequency as name_frequency,
    sectors.frequency as sector_frequency
FROM search_trends_symbol as symbols
JOIN companies ON companies.symbols = symbols.symbols
JOIN names ON names.symbols = companies.symbols AND names.timestamp = symbols.timestamp
JOIN names ON sectors.symbols = companies.symbols AND sectors.timestamp = symbols.timestamp

Essentially, all the matching data between the columns is lined up, and a new DataFrame is constructed from all of the combined data.

All of the NaN frequency values correspond to search queries which are too infrequent to be recorded by Google Trends, so they are replaced by 0. Now the search data is ready to be joined to the financial data.

import itertools

# Get a particular correlation for a pair of values
def corr(corrs, a, b):
    return corrs[a].apply(lambda x: x.loc[slice(None), b].sort_values(ascending=False))

correlations = (
    pd.DataFrame.from_records((
        {"symbol":symbol, "correlation": val, "financial_stat": a, "search_stat": b} 
        for a,b in itertools.product(
            ['price'],
            ['symbol_frequency', 'name_frequency', 'sector_frequency'],
        )
        for (symbol, _), val in corr(
                df.groupby('symbol').corr().groupby('symbol'),
                a,
                b,
            ).iteritems()
        ),
    )
    .set_index(["symbol", "financial_stat", "search_stat"])
)

The above essentially extracts all of the interesing correlations by extracting each pair and the placing each correlation coefficient into a series with a 3-level index of the form (symbol, stat1, stat2).

Conclusion
¶

In the exploration section, we identified a few potentially interesting correlations between company search trends and stock price. However, we were unsuccessful in creating a model that can effectively predict the market given solely this data. This is unsurprising. The stock market is incredibly complex and could not be reasonably expected to be modelled by such a simple formula. The high correlation values observed in NVDA are likely just artifacts of stock prices generally increasing over time. More data and a significantly more complex model is needed to accurately predict market movements. In fact, people make entire careers and multibillion dollar companies around the pursuit of better market models.
